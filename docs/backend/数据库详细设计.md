# 食光记数据库详细设计

## 1. 文档说明

本文档基于 `docs/需求文档v2.md`、`docs/backend/后端技术方案.md`、`docs/backend/后端开发TODO.md`，进一步细化 PostgreSQL 表结构、索引、外键、事务与迁移建议，作为 Prisma Schema 和 Migration 的直接依据。

设计边界：

- 当前优先支持 MVP，默认单家庭、单管理员。
- 所有主数据保留 `household_id`，为家庭协作预留。
- 数据库为 PostgreSQL 16+，ORM 默认 Prisma。

## 2. 全局约定

### 2.1 命名与类型

- 表名使用复数蛇形命名，如 `recipe_versions`。
- 主键统一 `id uuid default gen_random_uuid()`。
- 时间字段统一 `created_at`、`updated_at`、`deleted_at`。
- 业务日期使用 `date`，事件时间使用 `timestamptz`。
- 结构化快照统一使用 `jsonb`。
- MVP 阶段状态字段先使用 `varchar` + 代码枚举控制。

### 2.2 通用规则

- 软删除只用于 `recipes`、`categories`、`tags`、`moments`。
- 版本、菜单、购物清单等快照类数据不做软删除，优先保留历史。
- 所有受保护数据查询必须带 `household_id` 过滤。
- 部分唯一索引要忽略软删除记录。

### 2.3 核心枚举

| 字段 | 取值 |
| --- | --- |
| `households.status` | `active`, `archived` |
| `users.role` | `admin`, `member` |
| `users.status` | `active`, `invited`, `disabled` |
| `recipes.cover_source` | `custom`, `moment_latest`, `none` |
| `recipes.status` | `active`, `archived` |
| `media_assets.purpose` | `image` |
| `meal_plan_weeks.status` | `draft`, `finalized` |
| `meal_plan_items.meal_slot` | `lunch`, `dinner`, `extra` |
| `meal_plan_items.source_type` | `manual`, `random` |
| `random_pick_sessions.mode` | `single`, `week` |
| `random_pick_sessions.status` | `running`, `completed`, `abandoned` |
| `random_pick_results.decision` | `accepted`, `skipped`, `pending` |
| `shopping_lists.generated_from` | `manual`, `auto_refresh` |
| `shopping_lists.status` | `active`, `archived` |
| `shopping_list_items.item_type` | `ingredient`, `seasoning` |

## 4. 关键事务

### 4.1 创建菜谱

同一事务内完成：

1. 插入 `recipes`
2. 插入首个 `recipe_versions`
3. 插入步骤、食材、标签映射
4. 回写 `recipes.current_version_id`
5. 回写 `recipes.version_count = 1`

### 4.2 新建版本

同一事务内完成：

1. 读取当前最大 `version_number`
2. 生成新版本号
3. 写入版本及其子表
4. 生成 `diff_summary_text/json`
5. 回写 `recipes.current_version_id` 与 `version_count`

### 4.3 新增时光记录

同一事务内完成：

1. 插入 `moments`
2. 插入 `moment_images`
3. 回写 `recipes.latest_moment_at`
4. 回写 `recipes.latest_cooked_at`
5. 回写 `recipes.moment_count`

异步处理：

- 封面自动回填
- 图片缩略图或分享图

### 4.4 生成购物清单

同一事务内完成：

1. 读取目标周菜单
2. 聚合清单项
3. 生成新 `version_no`
4. 插入 `shopping_lists`
5. 批量插入 `shopping_list_items`

## 6. 实施注意事项

- Prisma 常规索引可写在 `schema.prisma`，部分唯一索引建议用 SQL migration 补充。
