## 默认约束

- [ ] MVP 以单家庭、单管理员优先，但所有核心数据表统一保留 `household_id`。
- [ ] 后端采用 `Next.js + TypeScript + PostgreSQL + Prisma` 单体架构，不拆微服务。
- [ ] 小程序主鉴权统一采用“`wx.login + code2Session + accessToken/refreshToken`”方案，不以 Cookie Session 作为默认链路。
- [ ] 业务规则统一放在服务层，前端不直接拼版本号、差异摘要、购物清单聚合、随机点菜规则。
- [ ] 输入输出统一采用 `Zod + DTO/Mapper` 管理，不直接暴露 Prisma 原始模型给前端。
- [ ] 菜谱版本必须独立建模，不能覆盖历史数据。
- [ ] 菜单项必须绑定具体 `recipe_version_id`，不能只绑定菜谱主表。
- [ ] 购物清单必须保存快照，不做纯实时计算结果页。
- [ ] 删除策略优先软删除，版本快照与购物清单快照优先保留历史。
- [ ] 本文档覆盖 PRD 全量能力，但不要求第一阶段一次性全部实现。
