# 产品需求文档（PRD）

## 家庭菜谱手账小程序 · 食光记

> 版本：V1.0 MVP  
> 日期：2026-03-23  
> 目标：单人版跑通核心闭环，支持后期扩展至多成员家庭

## 1. 产品概述

### 1.1 定位

一个面向家庭菜谱管理的轻量级手账工具，以菜谱版本管理和时间轴回忆为核心差异点，解决“同一道菜多次改良后忘记最佳配方”和“记录餐桌美好时刻”的痛点。

### 1.2 核心价值主张

- **版本可控**：一道菜的 N 种做法（山楂版 / 菠萝版糖醋排骨）完整留存，差异一目了然。
- **回忆可溯**：每道菜的历史时刻按时间聚合，点开即见“2024 年春节全家好评”的照片和评价。
- **计划可导**：点菜 → 生成菜单 → 导出购物清单（菜品 / 调料分离），简化备餐决策。

### 1.3 用户画像

| 角色 | 描述 | MVP 状态 |
| --- | --- | --- |
| 管理员（我） | 家庭菜谱主导者，录入、编辑、计划菜单 | ✅ 单人版即为此角色 |
| 成员 | 查看、点菜、评论、打分 | ⏸️ 二期开放邀请功能 |

## 3. 核心流程详解

### 3.1 菜谱录入流程（快速模式）

```text
入口：首页“+”或 菜谱库“新建”
├─ 必填：菜谱名称（如“糖醋排骨”）
├─ 选填：分类（下拉选择或新建）、标签（多选）
├─ 选填：主料（文本，如“排骨500g、山楂10颗”）
├─ 选填：步骤（文本列表，支持添加多条）
├─ 选填：小贴士（文本，如“山楂用新鲜的更酸爽”）
└─ 保存 → 创建 V1 版本 → 询问“是否记录这次做菜时刻？”
    ├─ 是 → 进入时光记录流程
    └─ 否 → 仅保存菜谱
```

### 3.2 版本管理流程

```text
场景：再次做糖醋排骨，改用菠萝
入口：菜谱详情页“新建版本”
├─ 复制 V1 内容作为草稿
├─ 修改：主料“山楂10颗”→“菠萝200g”、小贴士更新
├─ 系统自动生成差异摘要：“主料变化：山楂→菠萝；标签新增：果香”
├─ 保存为 V2
└─ V2 成为“当前版本”，V1 保留在历史中
```

**版本对比交互（MVP - 关键差异摘要）**

- 进入“版本历史”页面，列表展示所有版本。
- 点击任一版本，显示与当前版本的差异标签：
  - 主料变化：红色删除线 + 绿色新增
  - 标签变化：+ 新增 / - 移除
  - 步骤数变化：N 步 → M 步

### 3.3 时光记录流程（绑定菜谱）

```text
触发点：保存新版本时 或 菜谱详情页“记录时刻”
├─ 关联：自动绑定当前菜谱（可切换其他菜谱）
├─ 日期：默认今天，可修改（支持补记“2024年春节”）
├─ 内容：文字（故事/反馈）、照片（最多9张）
├─ 参与人：文本标签（如“全家”、“小明生日宴”）
├─ 打分：好吃程度⭐、难度⭐（可选）
└─ 保存 → 出现在该菜谱的“时光轴”顶部（最新在前）
```

**时光轴展示（按菜谱聚合）**

```text
糖醋排骨 详情页
├─ 当前版本（V2 菠萝版）
├─ [标签] 版本历史 | 时光轴
└─ 时光轴内容：
   2025-03-20  菠萝版首次尝试，小明说比山楂版更甜
   [图][图]  ⭐⭐⭐⭐⭐  ⭐⭐⭐

   2024-01-28  春节家宴，外婆传授山楂秘诀
   [图]  ⭐⭐⭐⭐⭐  ⭐⭐⭐⭐

   （点击展开查看更早记录）
```

### 3.4 点菜与购物清单流程

```text
入口：底部 Tab“点菜台”
├─ 本周菜单（默认空，从菜谱库添加）
│   ├─ 周一晚餐：糖醋排骨（V2 菠萝版）
│   ├─ 周二晚餐：番茄炒蛋
│   └─ ...支持拖拽排序
├─ 操作：生成购物清单
│   └─ 系统解析所有菜品的主料和调料字段
│       （MVP 简易规则：含“酱/油/盐/糖/醋/料酒/葱/姜/蒜”等关键词 → 调料清单，其余 → 菜品清单）
│   ├─ 菜品清单：排骨、菠萝、番茄、鸡蛋...（去重）
│   ├─ 调料清单：生抽、老抽、冰糖、醋...（去重）
│   └─ 支持手动勾选已购、编辑数量（文本）
└─ 导出：复制文本 或 生成图片分享
```

## 5. 关键页面原型描述

### 5.1 首页（时间轴 - MVP 简化版）

- 顶部：本周菜单快捷入口（显示“本周已规划 X 道菜”）。
- 主体：最新时光记录流（仅自己的记录，按时间倒序）。
- 卡片：菜谱封面图（或首图）+ 菜谱名称 + 日期 + 一句话预览。
- 点击 → 进入该菜谱详情页并定位到对应时光记录。

### 5.2 菜谱库页

- 顶部：分类 Tab（全部 / 肉菜 / 素菜 / 汤 / 点心）+ 标签筛选器。
- 列表：卡片网格 / 列表切换。
- 卡片：封面图、菜名、当前版本标签（如“V2 菠萝版”）、最近记录日期。
- 角标：时光记录数量（如有）。
- 搜索：按菜名搜索。

### 5.3 菜谱详情页

```text
顶部：大图（最新时光记录的首图 或 默认图）
信息区：菜名、分类标签、当前版本标识（V2）、创建时间
操作栏：[记一笔] [新建版本] [加入点菜]
Tab 切换：
├─ 做法（当前版本内容：主料/步骤/小贴士）
├─ 版本（列表：V2 菠萝版[当前]、V1 山楂版；点击对比）
└─ 时光轴（按时间倒序的所有记录，支持展开单条）
```

### 5.4 版本对比页（MVP）

- 顶部：当前版本 vs 选中版本标识。
- 差异摘要卡片（系统自动生成）：
  - 主料：- 山楂10颗 + 菠萝200g
  - 标签：+ 果香 - 酸甜
  - 步骤：5 步 → 4 步（简化收汁）
- 下方：两个版本的完整内容折叠展示（可选展开）。

### 5.5 点菜台页

- 周历视图：周一 ~ 周日，每餐一个卡片槽。
- 点击槽位 → 弹出菜谱库选择器（支持搜索 / 分类筛选）。
- 已选菜品显示缩略图 + 菜名，长按可删除或调整顺序。
- 底部固定按钮：“生成购物清单”。

### 5.6 购物清单页

- 分栏 Tab：菜品 / 调料。
- 清单项：名称 + 勾选框 + 数量备注（可编辑）。
- 操作：全选 / 反选、清空已购、复制文本、生成图片。

## 7. 技术建议

### 7.1 技术栈

- 前端：微信小程序原生框架（WXML + WXSS + JS）或 Taro（为后期跨端留余地）。
- 后端：云开发（微信云开发 / 阿里云 Serverless）降低初期成本。
- 数据库：云数据库（MongoDB / PostgreSQL）+ 本地 SQLite（离线支持）。

### 7.2 离线策略（关键）

```text
有网络时：
├─ 所有操作实时同步云端
├─ 全量数据缓存本地 SQLite
└─ 同步状态标记

无网络时：
├─ 读操作：直接读本地 SQLite
├─ 写操作：写入本地队列，标记“待同步”
└─ 恢复网络后：批量同步，冲突解决策略（时间戳优先或人工选择）
```

**关键优化**

- 菜谱库支持“预加载”（用户手动触发或 WiFi 自动同步）。
- 图片本地缓存 + 压缩上传。

### 7.3 版本差异算法（MVP 简化版）

```js
// 对比两个版本的文本字段，生成摘要
function generateDiffSummary(v1, v2) {
  const diffs = [];

  // 主料对比（简单文本匹配）
  if (v1.ingredients_text !== v2.ingredients_text) {
    diffs.push('主料调整');
  }

  // 标签对比
  const tagAdded = v2.tags.filter(t => !v1.tags.includes(t));
  const tagRemoved = v1.tags.filter(t => !v2.tags.includes(t));
  if (tagAdded.length) diffs.push(`新增标签：${tagAdded.join('、')}`);
  if (tagRemoved.length) diffs.push(`移除标签：${tagRemoved.join('、')}`);

  // 步骤数对比
  if (v1.steps.length !== v2.steps.length) {
    diffs.push(`步骤${v1.steps.length}步→${v2.steps.length}步`);
  }

  return diffs.join('；') || '细节微调';
}
```

## 8. 成功指标（MVP 验证）

| 指标 | 目标值 | 说明 |
| --- | --- | --- |
| 菜谱创建数 | ≥20 道 | 验证录入体验是否足够轻量 |
| 版本创建率 | ≥30% | 有多少菜谱产生了 V2+ 版本，验证版本功能价值 |
| 时光记录数 | ≥10 条 | 回忆功能的使用频率 |
| 点菜功能使用 | ≥5 次 / 周 | 核心闭环是否跑通 |
| 离线场景使用 | 无崩溃 + 数据不丢 | 技术稳定性验证 |

## 9. 风险与应对

| 风险 | 应对策略 |
| --- | --- |
| 版本对比算法过于简单，用户觉得没用 | MVP 先上，收集反馈后二期升级为行级对比 |
| 离线同步冲突（多设备同时改） | MVP 限制单设备，二期引入冲突解决 UI |
| 购物清单调料识别不准 | 允许用户手动拖拽调整分类，机器学习优化延后 |
| 图片存储成本 | 压缩策略（单图 < 500KB），初期云开发免费额度足够 |
